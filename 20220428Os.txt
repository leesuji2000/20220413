실제 운영체제의 이해 -> 실제 운영체제 구조 및 설계

쉘 종류: 사용자와 컴퓨터 하드웨어 또는 os간 인터페이스
-사용자의 명령을 해석해서, 커널에 명령을 요청하는 역할
-관련 시스템콜을 사용해서 프로그래밍이 작성되어 있다.
종류: bourne-again shell, bourn shell, c shell, korn shell

process management: 응용프로그램은 여러개의 프로세스로 관림
프로세스 스케줄러: 프로세스 실행, 종료 관리/ 인터럽트 처리 관리
메모리 매니지먼스: 가상메모리 - 페이지 기반 메모리 관리
IO 디바이스 매니지먼트: VFS, file, device drivers, network 관리

시스템 프로그램: 핵심은 쉘
	-bash(bourne - again shell)
	-내부는 시스템콜을 호출하도록 구현
각 프로그래밍 언어
	-필요시 해당 운영체제의 시스템콜 호출

안드로이드 스마트폰 내부구조
리눅스 커널 위에서 c라이브러리 존재(리눅스 커널 환경에서 쉘 가능, 리눅스 pc처럼 사용하거나 쉘(터미널) 리눅스 명령가능) -> 안드로이드는 자바 기반이기 때문에 런티임이라는 가상머신이 있다
-> 라이브러리와 런타임 위에 프레임워크(플랫폼)이 있고 그 위에 앱(응용프로그램)이 존재한다.
안드로이드는 크게 보면 리눅스 os위에 안드로이드 플랫폼이 있고 그 위에 응용프로그램 존재(엄밀히 말해서 안드로이드os가 아니라 안드로이드 플랫폼, 리눅스 os이다.)

IOT: 사물인터넷 - 각종 사물에 센서와 통신 기능을 내장하여 인터넷에 연결하는 기술
tiny os를 사용해서 초소형 운영체제를 사용한다.
